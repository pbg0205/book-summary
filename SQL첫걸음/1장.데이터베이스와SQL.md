# 1장.데이터베이스와 SQL

## 1. 데이터 베이스

### (1) 데이터와 데이터 베이스

- `데이터`는 컴퓨터 안에 기록되어 있는 숫자를 의미하고, `데이터 베이스`는 저장장치 내에 정리된 데이터의 집합을 말합니다.
- 데이터베이스 내의 데이터는 <u>**영구적으로 보존**</u>되어야 하므로 데이터베이서의 데이터는 HDD(하드디스크) 또는 SSD(플래시 메모리) 등 비휘발성 저장 장치에 저장합니다.

<br>

### (2) DB와 DBMS

- 데이터베이스 관리 시스템(`DBMS`) : 데이터 베이스를 효율적으로 관리하는 소프트웨어를 말합니다.
- DBMS의 장점
  - `생산성`
    - 시스템 구축 시, 기본 기능부터 구현할 필요없이 어떤 시스템에서든 데이터 검색, 추가, 삭제, 갱신을 처리할 수 있다.
  - `기능성`
    - 복수 유저 요청 대응, 대용량 데이터 저장 및 검색 기능을 제공하고, DB 관리 기능을 유저가 확장할 수 있어 유연한 시스템 개발이 가능합니다.
  - `신뢰성`
    - 많은 요청에 대응할 수 있고 이를 위해 하드웨어 여러 대로 구성하여 신뢰성을 높힘과 동시에 성능 향샹을 향상 시킬 수 있습니다.
    - 데이터베이스의 데이터를 다른 저장장치로 내보내거나(`export`), 다른 데이터베이스에 데이터를 집어넣는 기능(`import`)을 지원해 데이터 백업이 가능합니다.

<br>

### (3) SQL (Structured Query Language)

- 정의 : 관계형 데이터베이스를 조작할 때 사용하는 프로그래밍 언어
- 종류
  - `DDL` (Data Definition Language) : 데이터를 정의하는 명령어
    - ex) `CREATE`, `ALTER`, `DROP`, `TRUNCATE`
  - `DML` (Data Manipulation Language) : 데이터를 추가, 삭제, 갱신, 조작할 때 사용하는 명령어
    - ex) `SELECT`, `INSERT`, `UPDATE`, `DELETE`
  - `DCL` (Data Control Language) : 트랜잭션 제어 명령과 데이터 접근 권한을 제엉하는 명령을 포함
    - ex) `GRANT`, `REVOKE`, `COMMIT`, `ROLLBACK`

<br>

## 2. 다양한 데이터베이스

### (1) 데이터 베이스 종류
- `계층형 데이터베이스` : 폴더와 파일 등의 계층 구조로 데이터 저장 방식 데이터 베이스 ex) HDD, DVD
- `관계형 데이터베이스`  : 형과 열을 가지는 표 형식 데이터를 저장하는 형태의 데이터 베이스
- `객체지향 데이터베이스` : 객체 그대로를 데이터베이스의 데이터로 저장하는 데이터 베이스
- `XML 데이터베이스`
  - XML : 태그(ex. `<data>`)를 이용해 마크업 문서를 작성할 수 있게 정의한 것
  - XML 형식으로 등록된 데이터를 저장하는 데이터베이스
- `키-밸류 스토어(KVS)`
  - key:value 형태로 데이터를 저장하는 데이터 베이스
  - NoSQL(Not Only SQL)이라는 슬로건으로 시작하였으며, 열 지향 데이터베이스로 불립니다.

<br>

### (2) 데이터베이스 제품 (RDBMS)

- `ORACLE` : 가장 많은 시스템에서 사용 중인 RDBMS
- `DB2` : ORACLE처럼 오래된 RDBMS
- `SQL Server` : MS에서 개발한 RDBMS
- `PostgreSQL` : 오픈소스 RDBMS
- `MySQL` : 오프소스 RDBMS, 경량 데이터 베이스
- `SQLite` : 오픈소스 RDBMS, 임베디드 시스템에서 자주 사용

<br>

### (3) SQL 방언과 표준화

- SQL 방언 : 각각의 RDBMS 제품을 사용할 때 실행하는 명령어 차이
  - ex1) 키워드 생략
    - `Oracle`, `SQL Server` : DELETE FROM 포함 (X)
    - `PostgreSQL`, `MySQL` : DELETE FROM 포함 (O)
  - ex2) 외부 조인
    - 표준 SQL : **LEFT JOIN**
    - `Oracle` : +
    - `SQL Server` : *=
    
<br>

- 결론 : SQL 방언을 사용하는 것보다 **`표준 SQL`** 을 사용하도록 하자!

## 3. 데이터베이스 서버

### (1) Client / Server 모델

- 요청(Request)을 하는 입장을 **`클라이언트`**, 요청을 받아 응답(Response)을 처리하는 입장을 **`서버`** 라고 부른다.
- RDBMS는 사용자 별로 데이터 베이스 접근을 위해 **`사용자 인증`** 이 필요합니다. 만약 인증에 실패할 경우, 데이터베이스에 접근할 수 없습니다.
- RDBMS에 접속할 경우, 연결 상태를 유지해 재접속 없이 SQL 명령을 여러 번 보낼 수 있습니다.
- Client/Server가 네트워크로 연결되어 있다면, 서로 다른 머신에 두어도 무방하며 이는 처리를 분산시켜 <u>**시스템 전체 성능을 향상**</u> 시킵니다.

<br>

- **`루프 백`** 접속 : 클라이언트에서 서버에 접속 필요 시, 네트워크를 경유해서 PC의 서버로 되돌아와 접속하는 형태. 

<br>

### (2) CGI (Common Gate Interface)

- 웹 시스템은 Client/Sever 모델로 구성되며, 클라이언트가 서버에 요청하면 그에 상응하는 응답을 클라이언트에게 전달합니다.
- 동적인 HTML을 사용함은 데이터 처리가 필요하다는 것을 의미하고 이는 데이터베이스가 필요하며 데이터베이스와 접속하기 위해 `CGI프로그램`을 사용합니다. 
- 프로그래밍 언어로 만들어진 CGI는 <u>**DB서버와 접속하여 SQL 명령을 요청하는 클라이언트 역할**</u>을 합니다.
